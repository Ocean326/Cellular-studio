# Test Cases: 轨迹编辑模式 / 严格分段导出 / Studio 工作区布局

更新时间：`2026-04-27`

## A. 严格分段导出

### TC-A-001：互斥模式下保存后段列表不可重叠

- 前置：
  - 当前 reviewer 已设置
  - 打开互斥模式
- 步骤：
  1. 创建段 `A=(10,20]`
  2. 再创建段 `B=(15,30]`
- 预期：
  - 保存结果不出现重叠区间
  - 返回或回读的是 canonical 段列表

### TC-A-002：已有段内部新建段时起点吸附到旧段末尾

- 前置：
  - 已存在 `A=(10,20]`
  - 打开互斥模式
- 步骤：
  1. 创建新段，请求区间 `start=15, end=30`
- 预期：
  - 新段实际保存为 `(20,30]`

### TC-A-003：共享边界归属前一段

- 前置：
  - 存在两段 `(10,20]` 与 `(20,30]`
- 步骤：
  1. 导出 gps
  2. 检查 `timestamp=20`
- 预期：
  - 该点只导出一次
  - 标签归属于前一段

### TC-A-004：signal 记录跨边界时按段切分

- 前置：
  - 段列表为 `(10,20] road` 与 `(20,30] subway`
  - 原始 signal 记录为 `[15,25]`
- 步骤：
  1. 导出 signal
- 预期：
  - 导出结果拆成两条
  - 前半段标签为 `road`
  - 后半段标签为 `subway`

### TC-A-005：reviewer 段为空时 legacy fallback 语义可见

- 前置：
  - 当前 reviewer 无 segments
  - legacy `timeline_annotations/<uid>.json` 存在
- 步骤：
  1. 导出 segment label dataset
- 预期：
  - manifest 标出 `annotation_source=legacy`

- 预期：

- 视口：
  - `980x800`

## C. 轨迹编辑模式

### TC-C-001：单点拖动生成点 patch

- 前置：
  - 进入轨迹编辑模式
- 步骤：
  1. 选中一个点
  2. 拖动到新位置
- 预期：
  - 生成 point patch
  - 刷新后仍可回读

### TC-C-002：`Ctrl` 多选点后整体拖动

- 步骤：
  1. `Ctrl` 选中多个点
  2. 拖动其中一个点
- 预期：
  - 选择集整体平移
  - 相对位置保持不变

### TC-C-003：`Shift` 选段按 canonical order 生效

- 步骤：
  1. 选中锚点
  2. `Shift` 点击另一点
- 预期：
  - 两点之间的连续点被选中

### TC-C-004：右键批量修改元数据

- 步骤：
  1. 选中多个点
  2. 右键修改标签
- 预期：
  - 所有被选点的 metadata patch 一次性写入

### TC-C-005：编辑结果与 timeline 标注、review 结果隔离

- 步骤：
  1. 修改点位置
  2. 新增 timeline 段
  3. 保存整轨 review
- 预期：
  - 三类结果分开持久化
  - 任意一类回读失败不会污染另外两类

### TC-C-006：切换 reviewer 后轨迹修正互相隔离

- 前置：
  - Alice 已对 `UID=1001` 做过轨迹点拖动修正
  - Bob 尚未编辑该 UID
- 步骤：
  1. 当前 reviewer 从 Alice 切到 Bob
  2. 重新打开 `UID=1001`
- 预期：
  - Bob 视角看不到 Alice 的 point patch
  - 再切回 Alice 时，原修正可完整回读

### TC-C-007：刷新页面后 point patch 可回读并重新渲染

- 前置：
  - 已保存至少一个点的位置 patch 和一个 metadata patch
- 步骤：
  1. 刷新页面
  2. 重新进入同一 UID
- 预期：
  - 编辑模式关闭但 patch 已生效
  - 地图点位与标签状态按修正后数据渲染

### TC-C-008：`Shift` 连选基于 canonical order 而非临时渲染顺序

- 前置：
  - 当前时间窗口内发生过地图缩放或平移
- 步骤：
  1. 先选中一个锚点
  2. 调整地图缩放
  3. `Shift` 点击同层另一点
- 预期：
  - 选择区间仍按原始 canonical point order 连续
  - 不因 bucket、缩放或 tooltip 渲染顺序漂移

### TC-C-009：跨图层多选允许整体拖动，但禁止混层批量改标签

- 步骤：
  1. `Ctrl` 选中两个不同图层的点
  2. 拖动其中一个点
  3. 再右键尝试批量修改标签
- 预期：
  - 多点拖动仍对全部选中点生效
  - 右键菜单明确提示“仅支持同层批量改标签”或禁用该操作

### TC-C-010：无 reviewer 身份时不可进入轨迹编辑模式

- 步骤：
  1. 清空当前 reviewer session
  2. 点击“轨迹编辑”开关
- 预期：
  - 不进入编辑态
  - 页面提示先设置标注者身份

### TC-C-011：顶部模式切换与编辑工作台联动

- 步骤：
  1. 进入一个可编辑 UID
  2. 点击顶部中部的“编辑”
  3. 再点击“批注”
- 预期：
  - 顶部 mode switch 高亮同步切换
  - 编辑模式显示保存/撤销/重做/显示经纬度等编辑专属控件
  - 切回批注后恢复 review 控件，编辑草稿不丢失

### TC-C-012：撤销/重做回放单次轨迹编辑事务

- 步骤：
  1. 进入编辑模式
  2. 拖动一个点或批量修改一组点标签
  3. 点击“撤销”
  4. 再点击“重做”
- 预期：
  - 撤销后 point patch 数量或点位/标签状态回到变更前
  - 重做后恢复到变更后状态
  - 撤销栈、重做栈按钮禁用态与当前历史状态同步

### TC-C-013：显式保存与另存为 JSON

- 步骤：
  1. 在编辑模式下制造至少一个未保存修改
  2. 点击“保存”
  3. 再展开保存菜单，点击“另存为 JSON”
- 预期：
  - 保存后状态文案更新为已保存，未保存提示消失
  - 再次点击保存主按钮应处于禁用或提示“无需重复保存”
  - 浏览器触发 JSON 下载，文件名包含 UID 与 reviewer

### TC-C-014：空格按住时才切回地图平移/缩放

- 步骤：
  1. 进入编辑模式
  2. 不按空格直接拖动地图区域，确认优先响应点编辑
  3. 按住空格后拖动地图并滚轮缩放
  4. 松开空格
- 预期：
  - 未按空格时地图平移/缩放被锁住
  - 按住空格时地图可平移/缩放，状态提示同步变化
  - 松开空格后重新锁回编辑态地图交互

### TC-C-015：拖回基线后 patch 自动收敛并清除脏状态

- 前置：
  - 当前轨迹尚未保存新的轨迹修正
- 步骤：
  1. 拖动一个点或一组选中的点，制造未保存 patch
  2. 再把这些点精确拖回原始位置
- 预期：
  - `pointPatches` 中对应 patch 自动删除
  - 未保存标记消失
  - “保存”按钮重新禁用

### TC-C-016：保存失败时保留本地草稿，不丢编辑上下文

- 前置：
  - 当前已有至少一个未保存轨迹修正
  - 模拟 `/api/track-edits` POST 失败
- 步骤：
  1. 点击“保存”
- 预期：
  - 页面提示保存失败
  - 当前 patch 仍保留在本地
  - `dirty` 仍为 true
  - 用户可继续编辑、重试保存或另存为 JSON

### TC-C-017：加载失败时回退到本地 reviewer/UID 草稿

- 前置：
  - 当前 reviewer/UID 在本地已有未成功上送的轨迹修正
  - 模拟 `/api/track-edits` GET 失败
- 步骤：
  1. 重新打开同一 UID
- 预期：
  - 页面仍能恢复该 reviewer/UID 的本地 patch
  - 不会因为服务端读取失败而把本地草稿清空

### TC-C-018：跨 reviewer 切换时轨迹修正严格隔离

- 前置：
  - Reviewer A 对同一 UID 已有未保存或已保存轨迹修正
  - Reviewer B 尚未编辑该 UID
- 步骤：
  1. 从 Reviewer A 切换到 Reviewer B
  2. 打开同一 UID
  3. 再切回 Reviewer A
- 预期：
  - Reviewer B 视角看不到 Reviewer A 的轨迹修正
  - 切回 Reviewer A 时原修正完整恢复

### TC-C-019：右键批量改标签遇到混合值时给出明确占位引导

- 前置：
  - 同一图层已选中多个点，且标签值不一致
- 步骤：
  1. 右键打开批量编辑菜单
- 预期：
  - 菜单标题/副标题明确提示“当前值不一致”
  - 下拉框默认不直接选中任一值
  - Apply 按钮在未重新选择统一值前保持禁用
