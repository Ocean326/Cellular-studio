# 分段标签数据接口说明

## 1. 交付内容

每个 `uid` 交付两个 CSV 文件：

- `{uid}_signal.csv`
- `{uid}_gps.csv`

示例：

- `1001_signal.csv`
- `1001_gps.csv`

## 2. 文件组织形式

推荐按 UID 组织：

```text
mock_dataset/
  1001_signal.csv
  1001_gps.csv
  1002_signal.csv
  1002_gps.csv
```

也可以按子目录组织：

```text
mock_dataset/
  1001/
    1001_signal.csv
    1001_gps.csv
  1002/
    1002_signal.csv
    1002_gps.csv
```

约束：

- 一个 `uid` 对应一对文件
- 一个文件内只包含一个 `uid`
- 文件内容按时间升序排列

## 3. 标签枚举

`status` 字段统一使用以下枚举值：

| status | 中文说明 |
| --- | --- |
| `subway` | 地铁 |
| `road_car` | 驾车 |
| `road_bus` | 公交 |
| `road_taxi` | 出租车 |
| `road` | 泛道路乘车 |
| `low_speed` | 低速 |
| `train` | 火车 |
| `stay` | 驻留 |
| `flight` | 飞机 |

说明：

- 能细分时，优先使用 `road_car / road_bus / road_taxi`
- 无法细分但可以确定是道路乘车时，使用 `road`

## 4. `{uid}_signal.csv` 接口

### 字段定义

| 字段名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `uid` | string | 是 | 用户或轨迹 ID |
| `cid` | string | 是 | 信令点或基站标识 |
| `longitude` | float | 是 | 经度 |
| `latitude` | float | 是 | 纬度 |
| `t_in` | int64 | 是 | 开始时间，Unix 毫秒时间戳 |
| `t_out` | int64 | 是 | 结束时间，Unix 毫秒时间戳 |
| `status` | enum string | 是 | 该条信令记录对应的分段标签 |

### 示例

```csv
uid,cid,longitude,latitude,t_in,t_out,status
1001,cell_a,116.39731,39.90812,1710000000000,1710000180000,stay
1001,cell_b,116.40152,39.90746,1710000180000,1710000480000,subway
1001,cell_c,116.41783,39.90511,1710000480000,1710000780000,subway
1001,cell_d,116.43622,39.90345,1710000780000,1710001080000,road_taxi
```

## 5. `{uid}_gps.csv` 接口

### 字段定义

| 字段名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `uid` | string | 是 | 用户或轨迹 ID |
| `longitude` | float | 是 | 经度 |
| `latitude` | float | 是 | 纬度 |
| `timestamp` | int64 | 是 | 点时间，Unix 毫秒时间戳 |
| `status` | enum string | 是 | 该点对应的分段标签 |

### 示例

```csv
uid,longitude,latitude,timestamp,status
1001,116.39710,39.90805,1710000000000,stay
1001,116.39712,39.90804,1710000060000,stay
1001,116.40130,39.90750,1710000240000,subway
1001,116.41755,39.90520,1710000540000,subway
1001,116.43590,39.90360,1710000900000,road_taxi
```

## 6. 通用约束

- 时间字段统一为 Unix 毫秒时间戳
- 坐标字段统一为十进制度经纬度
- `signal` 中要求 `t_out >= t_in`
- 同一连续时间段内，`gps` 与 `signal` 的 `status` 应保持一致

## 7. 建议校验项

- 文件名中的 `uid` 与文件内容中的 `uid` 一致
- 每个文件只包含一个 `uid`
- 时间顺序非降序
- `status` 只能取约定枚举值
