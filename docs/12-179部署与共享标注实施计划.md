## 3. Selected Path

本轮选择：

- `existing feature iteration + research-backed implementation`

原因：

- 基础标注能力已存在
- 当前缺口集中在“运维化、流程化、导出化、接入化”
- 不需要推翻现有 reviewer namespace 架构

## 4. Phase Plan

## Phase 1：推送现阶段代码 + 179 基础部署

### 目标

- 固化 179 推荐目录结构
- 准备 `app/current`
- 准备 `datasets/`
- 准备 `shared/published`、`shared/incoming`、`shared/exports`
- 用 `systemd + nginx` 起服务

### 产物

- `scripts/init_179_server_layout.py`
- `deploy/systemd/trajectory_annotation_studio.service.example`
- `deploy/nginx/trajectory_annotation_studio.nginx.conf.example`
- 可能的 `scripts/print_179_deploy_steps.py` 或等价说明

### 验收

- 脚本可在本地 dry-run 生成目标目录结构
- 模板能表达：
  - localhost bind
  - `--batches-root`
  - nginx 反代
  - 单实例运行约束

## Phase 2：发布第一轮 `10天 * 10000 UID`

### 目标

- 在 `datasets` 准备完整 `signal100_10d_uid10000_round1/result`
- 切成 `10` 个 `1000 UID` shard
- 在 `published` 建 `10` 个批次入口
- 每个 shard 通过 `source_result_root` 挂到 studio

### 依赖

- Phase 1 目录和模板完成
- Lenovo 运算结果由 operator 推到 179

### 产物

- `scripts/validate_published_batch.py`
- `scripts/publish_server_batch.py`
- 可选 `scripts/split_result_manifest_shards.py`

### 验收

- 对一个本地样例批次可完成：
  - validate
  - publish
  - `/api/batches` 可见

## Phase 3：补导出闭环

### 目标

- 增加 `reviewer bundle export`
- 把“我的标注 + 对应原数据”打成可下载包

### 推荐首发范围

- 样本级携带：
  - `raw.csv`
  - `snap.csv`
  - `od.csv`
  - `fmm.csv`
  - `line.csv`
  - `review.json`
  - `timeline_annotations.json`
- 元信息：
  - `bundle_manifest.json`
  - `source_manifest.json`

### 产物

- `web/review_lib.py` 扩展 bundle export 能力
- `web/export_reviewer_bundle.py`
- 对应测试

### 验收

- 一个 reviewer 可导出自己的全部已标注子集
- 输出目录或 zip 结构稳定

## Phase 4：补上传接入闭环

### 目标

- 增加 `incoming -> validate -> publish` 脚本
- 做 Web 上传功能

### 首发边界

首发只做：

- 上传 `.zip`
- 服务端写入 `incoming`
- 基础结构校验
- operator 侧 CLI 解包、校验并按需发布
- 列出 incoming 条目

首发不做：

- 浏览器端复杂打包目录树
- 细粒度权限审批系统
- 多文件断点续传

### 产物

- `web/review_server.py` upload/admin endpoints
- `web/admin_upload.html`
- `scripts/process_incoming_upload.py`
- 对应测试

### 验收

- 可以通过 Web 上传一个 zip 到 `incoming`
- 服务端能列出上传结果
- operator 能把校验通过的样例发布成批次

## 7. Main Risks

- 179 当前网络路径可能并不从本机直达
- `review_server.py` 目前适合单实例，不适合多进程并发写共享目录
- Web 上传首发若做得过重，会拖慢整体落地
- Lenovo -> 179 的数据推送链路依赖 operator 环境而不是当前本机
