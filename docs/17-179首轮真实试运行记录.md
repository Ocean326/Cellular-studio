## 5. 本轮暴露出的流程问题

### 5.1 文档原先默认假设有 sudo

这在真实 179 上不成立。

改进：

- 14/15/16 已补充 `user-space pilot` 路径
- 把 `/srv` 正式姿态和 `/home/<user>` 试运行姿态分开写

### 5.2 agent 侧访问方式原先写得过于理想

首轮 `pilot` 没有域名和 nginx，agent 不可能直接访问远端 HTTP。

改进：

- 在快速接入指南里补了 `ssh -L` 访问方式
- 明确 `pilot` 阶段是“访问方式受限”，不是“接入合同变化”

### 5.3 shard 准备责任必须写得更明确

`publish_server_batch.py` 不会帮你自动把全量结果拆成 10 个 reviewer-facing shard。

改进：

- 详细操作手册已明确要求 operator 先准备独立的 shard `result root`

### 5.4 有些步骤不能并行

首轮试运行里出现过一个“看起来像 bug”的假象：

- `review` 返回成功
- 紧接着并行触发 `reviewer bundle export`
- 导出结果出现 `sample_count = 0`

根因不是代码错误，而是执行顺序错了。

改进：

- `publish -> validate`
- `review -> export`

这两类步骤都明确要求串行，不要并行触发
