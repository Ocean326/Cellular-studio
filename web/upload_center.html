使用说明

这页是最小前端壳，默认按你提供的 API 契约工作。

推荐流程

  1. 选择类型、可见性、标注模式，并提交原始文件。
  2. 在“我的上传”中确认状态，再按需触发处理。
  3. 如果后端返回了 batch 名称，就可以直接打开 studio。

状态提示

  • uploaded / created：记录已创建，通常表示文件已提交或等待下一步。
  • queued / validating / processing：后端正在接手处理。
  • published / completed：可尝试直接打开对应 batch。
  • failed / error：查看错误摘要，必要时删除后重传。
