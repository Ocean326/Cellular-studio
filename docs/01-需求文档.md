# 标注工具系统升级需求文档

## 1. 背景

当前 `cellular_quality` 里的标注页已经具备一些 organizer 可用能力：

- 批次切换 (`batch switch`)
- 整轨审核 (`accept / reject / skip`)
- 结果查看与多图层展示
- 审核写回与 `accepted_assets` 导出

但它仍然更像“审核附页”，还不是一个独立、可长期演进的标注系统。现阶段主要瓶颈有四类：

1. 工程边界不清晰：前端、审核逻辑、结果导出仍挂在 `cellular_quality` 内，后续多人改功能容易互相干扰。
2. 数据模型不够：现在默认一条 UID 对应一条轨迹，不适合“同一人跨多天、任意时段”的连续审阅。
3. 标注表达不足：目前只有整轨 `accept / reject / skip`，缺少二级标签与时间段标签。
4. 交互深度不够：没有时间窗口选择器、没有时间轴、没有分段创建与编辑能力。

## 2. 目标

把现有标注页升级为一个独立子项目 `trajectory_annotation_studio`，并在 v1 达成以下目标：

1. 支持同一人跨多天长轨迹的窗口化查看。
2. 保留整轨审核，并支持 `accept` 之后的二级自定义标签。
3. 支持基于时间轴的分段标注。
4. 支持项目级标签字典与批次级启用控制。
5. 支持本地独立 Git 仓库与 `git worktree` 协作。

## 3. 产品边界

### 3.1 v1 要做

- 仓内独立子项目
- 复用 `cellular_quality` 上游结果
- 支持多天长轨迹
- 支持时间段选择器
- 支持整轨审核与二级标签
- 支持时间轴分段标注
- 支持 organizer 导出已审核与已标注资产

### 3.2 v1 不做

- 云端部署
- 鉴权与权限系统
- 多用户实时协同
- agent 自动预标
- 正式平台化或 SaaS 化

## 4. 核心用户与使用方式

### 4.1 Organizer

主要负责：

- 导入批次
- 审核轨迹
- 建立标签字典
- 管理导出资产
- 给 `research_arena` 或其他研究任务提供可用的参考标注

### 4.2 开发者 / Agent

主要负责：

- 基于独立仓库开发新功能
- 使用 `worktree` 并行修改前端、后端或数据适配层
- 不直接改变上游 `cellular_quality` 的主流程语义

## 5. 数据与对象模型

### 5.1 上游依赖

上游仍以 `cellular_quality` 为主，继续沿用它的处理管线和结果组织方式。

多天长轨迹的数据源以 Lenovo `E:` 盘中的 `signal_100+cid2gps.csv` 为主输入，由兼容管线整理为本项目可消费的轨迹与图层结果。

### 5.2 主对象

- `track`
  一个人跨多天的一条完整长轨迹。
- `batch`
  一批待审任务，可包含若干 `track`。
- `review`
  整轨审核结果，包含 `accept / reject / skip`、备注与二级标签。
- `segment annotation`
  时间段标签，绑定到某条 `accept` 轨迹的某个区间。
- `label catalog`
  项目级标签字典。

### 5.3 审核语义

一级审核固定为：

- `accept`
- `reject`
- `skip`

二级标签与分段标签只允许挂在 `accept` 轨迹下。

如果一条轨迹从 `accept` 被改成 `reject` 或 `skip`：

- 历史分段标签不删除
- 但会被标记为 `inactive`
- 不参与导出与后续正式使用

## 6. 交互要求

### 6.1 页面布局

- 左侧：批次、轨迹列表、筛选器、整轨审核区、标签摘要
- 左上角地图控件下：图层控制
- 地图正下方：时间段选择器 + 时间轴

### 6.2 时间段选择器

作用：

- 控制当前地图与明细区展示哪个时间窗口
- 支持输入
- 支持滚动调整
- 支持跨日窗口

### 6.3 时间轴与分段标注

主交互固定为：

1. 选择当前锚点图层
2. 通过时间轴移动光标
3. 右键创建起点
4. 拖动到终点时刻
5. 右键创建终点
6. 选择标签并保存

时间轴锚点允许切换当前图层，例如：

- `raw`
- `snap`
- `od`
- `line`

### 6.4 标签要求

项目级标签字典分为两类：

- `trajectory_tags`
- `segment_tags`

每个标签至少包含：

- 内部 ID
- 显示名称
- 颜色
- 描述
- 是否启用

批次只能控制“启用哪些标签”，不能各自定义一套互不兼容的新标签体系。

## 7. 输出与导出

系统需要支持三类导出：

1. 整轨审核结果
2. `accept` 样本的参考资产
3. 带轨迹标签与分段标签的标注资产

导出要求：

- 任意时刻都可以导出一部分已完成结果
- 导出结果必须可回溯到批次、轨迹、审核人和时间

## 8. 成功标准

当以下条件满足时，视为 v1 需求达成：

1. 新工具能独立启动，不依赖旧页面继续挂在 `cellular_quality/src/web`
2. 同一人跨多天轨迹能以时间窗口方式查看
3. 整轨审核、二级标签、分段标签都可写回
4. `accept` 与细标注的语义边界清晰
5. 可导出当前批次中任意已完成的标注资产
6. 仓库可正常使用 `git worktree` 进行并行开发
